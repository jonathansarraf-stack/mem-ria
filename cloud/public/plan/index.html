Cortex Business Plan

Persistent memory for AI agents — Analise de viabilidade, precificação e go-to-market
Gerado: 17 abril 2026 · v1.0

1. O Produto

Cortex (mem-ria) dá memória persistente pra AI agents. O dev instala via npm i mem-ria, configura como MCP server no Claude Code, e o AI começa a lembrar entre conversas. Brain cycle noturno computa salience, poda ruído, consolida conhecimento.

CLI + MCP Server

Roda local na máquina do dev. brain.db em SQLite. Zero dependência cloud.

Cloud Dashboard

Assinante faz login, vê brain visualmente: memórias, entidades, saúde, grafo.

Brain Sync

mem-ria sync comprime e sobe brain.db pro cloud. Dashboard atualiza.

Stack: TypeScript, Hono, SQLite (better-sqlite3), JWT + OTP 2FA, Stripe billing, PM2, Nginx.

7. Go-to-Market

Fase 1: Validação (mês 1-2) — 0→50 users

  • Publicar no npm (já feito: mem-ria v0.2.0)
  • Post no r/ClaudeAI, r/ChatGPTCoding, Hacker News (Show HN)
  • Thread no X/Twitter com demo (CLI → dashboard em 2 min)
  • Claude Code community (Discord, GitHub discussions)
  • Meta: 500 installs npm, 20-30 free users ativos, 5 pagantes

Fase 2: Traction (mês 3-6) — 50→500 users

  • Blog posts: "How I gave Claude Code a brain", "Why your AI forgets everything"
  • Integrações: Cursor, Windsurf, Continue.dev (MCP é universal)
  • Product Hunt launch
  • GitHub stars como social proof (README polido, GIFs de demo)
  • Meta: 5K installs, 200 free users, 50 pagantes ($900 MRR)

Fase 3: Growth (mês 6-12) — 500→2K users

  • Content marketing: comparativos (Cortex vs Mem0 vs Zep)
  • Partnerships: AI tools newsletters, dev tool roundups
  • Team plan: vender pra pequenas equipes de desenvolvimento
  • Meta: 50K installs, 1K free, 200 pagantes ($3.5K MRR)
